Metodologia de estudo — Para cada tópico: tente escrever do zero → consulte documentação → use Claude só para revisar ou destravar. Anki para conceitos que precisam ficar automáticos (window functions, explain plan, Delta configs). Não avance de fase sem conseguir executar o tópico anterior sem consultar.

Projetos âncora — onde o aprendizado vira prova

🏪
Renner — silver/gold autoral
Criar uma camada silver e gold do zero, sem template — só você e a documentação. Escolha uma métrica real do seu vertical (cobertura de estoque, mix de categoria). O código vai para revisão do time.
🏃
Garmin pipeline completo
Pipeline Bronze→Silver→Gold com dados reais de corrida. Inclui ingestão via API, orquestração, qualidade de dados e camada analítica. Você decide a arquitetura — sem copiar padrão pronto.
🧪
Suite de testes para pipeline
pytest cobrindo a lógica de transformação de pelo menos uma camada silver. Inclui fixtures com dados sintéticos, testes de borda (nulos, duplicatas, schema), e integração com CI.
📄
ADRs de decisão técnica
Documento de decisões arquiteturais das escolhas que você fez nos projetos. "Por que usamos DLT aqui e não Workflow?", "Por que incremental e não full?". Especialista explica o porquê.
📊
Sessão técnica no time
Apresentar algo que você aprendeu profundamente para o time — window functions avançadas, otimização de Delta, padrões de pipeline. Ensinar é o maior teste de domínio real.
🔄
Refatoração com critério (mês 12)
Pegar algo que você escreveu no mês 1–2 e reescrever com o que aprendeu. Ver a diferença entre o "você de antes" e o especialista que você se tornou. Isso é evidência real de evolução.

Princípio central — O projeto paralelo (Garmin/OCI) existe para você tomar decisões que na Renner já estão definidas: governança, acesso, arquitetura, ferramentas. Lá você projeta, escolhe, erra e aprende o que o ambiente protegido não deixa. O que você aprende lá alimenta o pensamento que você traz para cá.
