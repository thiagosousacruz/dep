Do executor
ao especialista.

12 semanas para dominar SQL, PySpark, Databricks e ADF — resolvendo problemas reais de negócio, de ponta a ponta.

12 Semanas
6 Fases
1 Objetivo
// PRINCÍPIO

Cada coisa que você aprende precisa aparecer em algo real na Renner na semana seguinte. Isso é o que transforma conhecimento em competência. Não separe estudo de trabalho.

Fase 01 · Semanas 1–2
SQL — Fundação Sólida
Você aprende a linguagem que vai usar todo dia, para sempre.
Semana 01
Leitura e filtros
  • SELECT, FROM, WHERE, ORDER BY, LIMIT
  • Operadores: =, !=, >, <, BETWEEN, IN, LIKE
  • Lógica booleana: AND, OR, NOT
  • Aliases com AS
Critério de avanço

Consultar qualquer tabela, filtrar e ordenar sem olhar documentação.

Semana 02
Agregações e JOINs
  • GROUP BY + COUNT, SUM, AVG, MAX, MIN
  • HAVING para filtrar grupos
  • INNER JOIN e LEFT JOIN — quando usar cada um
  • RIGHT JOIN e FULL OUTER JOIN
  • JOINs com múltiplas condições
Critério de avanço

Cruzar duas tabelas e agregar o resultado com confiança.

// como praticar

Use dados reais da Renner ou o ambiente NovaMed. Escreva queries que respondam perguntas de negócio reais — "quais categorias venderam mais essa semana?" — não exercícios artificiais.

Fase 02 · Semanas 3–4
SQL — Intermediário + Pensamento Analítico
Você começa a resolver problemas que seu gestor resolve hoje.
Semana 03
Subqueries e lógica condicional
  • Subqueries no WHERE e no FROM
  • CTEs com WITH ... AS — organizar queries longas
  • CASE WHEN para lógica condicional
  • Tratamento de nulos: COALESCE, NULLIF, IS NULL
Critério de avanço

Escrever uma query com CTE e CASE WHEN sem consultar nada.

Semana 04
Window functions
  • ROW_NUMBER, RANK, DENSE_RANK
  • LAG e LEAD — comparar linha com anterior/próxima
  • SUM OVER e AVG OVER — acumulados e médias móveis
  • PARTITION BY — janelas por grupo
  • Datas: DATEDIFF, DATEADD, FORMAT
Critério de avanço

Calcular variação percentual, ranking por categoria e média móvel sem ajuda.

// como praticar

Pegue relatórios que existem hoje na Renner e tente recriá-los do zero em SQL. Não peça a query — entregue o resultado e valide com o original.

Fase 03 · Semanas 5–6
Databricks na prática + leitura de notebooks
Você para de ter medo do ambiente e começa a se sentir em casa.
Semana 05
Orientação no ambiente
  • Como o Databricks funciona: clusters, notebooks, células
  • Spark SQL dentro do Databricks com spark.sql()
  • Leitura de Delta Tables: spark.read.format("delta")
  • Entender a arquitetura bronze/silver/gold sem precisar construir
Critério de avanço

Rodar uma query Spark SQL em um notebook existente sem erros.

Semana 06
PySpark básico — leitura e inspeção
  • df.select(), df.filter(), df.groupBy(), df.withColumn()
  • Como inspecionar dados: df.show(), df.printSchema(), df.describe()
  • Como salvar resultado: df.write.format("delta").save()
  • Leitura de notebooks existentes: entender bloco por bloco
Critério de avanço

Abrir um notebook desconhecido e comentar o que cada bloco faz.

// como praticar

Pegue notebooks existentes no ambiente da Renner, leia linha por linha e escreva em comentário o que cada bloco faz. Depois tente reescrever partes menores do zero.

Fase 04 · Semanas 7–8
PySpark intermediário + resolução de problemas reais
Você começa a entregar de ponta a ponta, não só executar.
Semana 07
Transformações e joins
  • Joins no PySpark: df.join(other, on=..., how=...)
  • Funções úteis: F.when(), F.coalesce(), F.lit()
  • Tratamento de dados sujos: nulos, tipos errados, duplicatas
  • Como ler arquivos externos: CSV, Parquet, Excel
Critério de avanço

Cruzar duas tabelas Delta e salvar o resultado transformado.

Semana 08
Window functions no PySpark
  • Window.partitionBy() e Window.orderBy()
  • F.lag(), F.lead(), F.rank(), F.row_number()
  • SUM e AVG acumulados com janela
  • Combinando PySpark com Spark SQL no mesmo notebook
Critério de avanço

Entregar uma tabela final correta explicando as decisões tomadas no caminho.

// como praticar

Pegue uma demanda real, mesmo que pequena, e proponha ao seu gestor que você faz a entrega. Mostre o resultado antes de mostrar o código.

Fase 05 · Semanas 9–10
ADF + ingestões como seu gestor faz
Você entende como os dados chegam e consegue criar ou adaptar pipelines.
Semana 09
ADF — fundação e ingestões básicas
  • Como um pipeline no ADF funciona: atividades, fluxo, parâmetros
  • Ingestão de arquivos flat: CSV, Excel do SharePoint para bronze
  • Como parametrizar pipelines para reusar em múltiplos contextos
  • Tratamento de erros básico: If Condition, On Failure
Critério de avanço

Ler e entender qualquer pipeline existente na Renner sem ajuda.

Semana 10
Ingestões via API e SFTP
  • Ingestão via API REST: linked service, headers, autenticação
  • Ingestão via SFTP: configurar, pegar arquivo, mover para Data Lake
  • Criar um pipeline novo do zero — mesmo que cópia de existente
  • Debug e monitoramento de execuções no ADF
Critério de avanço

Criar uma ingestão nova sem precisar que o gestor configure do zero pra você.

// como praticar

Olhe os pipelines existentes na Renner com objetivo de entender, não de mudar. Depois proponha criar um pipeline simples do zero — mesmo que seja uma cópia de algo que já existe.

Fase 06 · Semanas 11–12
Domínio de negócio + entregas autônomas
Você vira referência nos problemas que importam para a Renner.
Semana 11
Mapeamento dos problemas recorrentes
  • Identificar os 5–6 problemas que mais aparecem no time
  • Cobertura de estoque, vendas por categoria, ruptura, giro de produto
  • Entender a lógica de negócio por trás de cada problema, não só a query
  • Construir repositório pessoal de queries e notebooks comentados
Critério de avanço

Ter pelo menos 3 soluções documentadas para problemas reais do time.

Semana 12
Autonomia total — do pedido à entrega
  • Receber o pedido, entender a dor, entregar a tabela
  • Documentar o que foi feito: um parágrafo explicando a lógica
  • Propor soluções para problemas que o gestor ainda não resolveu
  • Ser a referência técnica que outros consultam
Critério de avanço

Seu gestor te passa um problema novo e você volta com uma proposta de solução, não com uma pergunta.

// como praticar

Pegue demandas de ponta a ponta. Documente tudo que você fez — um parágrafo explicando a lógica é o que separa um executor de um especialista.
